# Maintainers

This file lists who maintains Tarit and what they look after. Code review
ownership is enforced through [.github/CODEOWNERS](.github/CODEOWNERS).

## Current maintainers

| Name           | GitHub           | Areas |
|----------------|------------------|-------|
| Abhishek Anand | [@abhishek-anand](https://github.com/abhishek-anand) | Everything: `vmm/` (the VMM), `orch/` (taritd), `proto/` (wire protocol), CI, releases, security |

Tarit is a young project with a single maintainer. As the contributor base
grows, this table will grow with it.

## Responsibilities

Maintainers:

- review and merge pull requests,
- triage issues and security reports (see [SECURITY.md](SECURITY.md)),
- cut releases and keep [CHANGELOG.md](CHANGELOG.md) accurate,
- keep CI, dependency policy (`deny.toml`), and supply-chain checks healthy.

## Becoming a maintainer

See [GOVERNANCE.md](GOVERNANCE.md). The short version: sustained,
high-quality contributions in an area, then an invitation from the existing
maintainers.
